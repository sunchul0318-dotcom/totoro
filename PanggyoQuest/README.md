# 판교 퀘스트 (Pangyo Quest)

> 지하철 **판교역**에서 **위메이드플레이**까지 — 출근길의 빌런들을 무찌르고 정시 출근을 사수하라!
> 드래곤 퀘스트를 오마주한 레트로 감성의 **실시간 액션 RPG**.

Next.js(App Router) + TypeScript + Tailwind CSS + HTML5 Canvas 로 만든, **외부 게임 엔진·유료 에셋 없이** 순수 코드로 완성한 웹 게임입니다. 모든 그래픽은 Canvas 드로잉·도형·이모지·CSS 로 실시간 생성됩니다.

```bash

```

프로덕션 빌드:


### Cloudflare Pages 배포

`@cloudflare/next-on-pages` 파이프라인을 사용한다. Cloudflare Pages 프로젝트 설정:

- **빌드 명령어**: `npx @cloudflare/next-on-pages@1`
- **빌드 출력 디렉터리**: `.vercel/output/static`
- **호환성 플래그**: `nodejs_compat`

리포지토리에 이 값들을 고정해 두었으므로 대시보드 변경 없이 동작한다:

- [`.npmrc`](.npmrc) — `legacy-peer-deps=true` (next-on-pages 설치 시 peer 의존성 충돌 방지)
- [`wrangler.toml`](wrangler.toml) — `pages_build_output_dir` + `nodejs_compat`
- `next.config.mjs` — next-on-pages 호환을 위해 `output: 'export'` 미사용

권장 환경: 최신 Chrome/Edge/Safari. 데스크톱 키보드 및 모바일 터치 모두 지원.

## 🎮 조작법

| 동작 | 키보드 | 모바일 |
|------|--------|--------|
| 이동 | `W A S D` / 방향키 | 좌측 가상 조이스틱 |
| 공격 (콤보) | `J` / `Z` / `Space` | 우측 **공격** 버튼 |
| 회피 구르기 (무적) | `K` / `X` / `Shift` | 우측 **회피** 버튼 |
| 필살기 (SP 100 소모) | `L` / `C` | 우측 **필살** 버튼 |
| 일시정지 | `ESC` / `P` | 우상단 ❚❚ 버튼 |
| 대사 진행 | `Enter` / `Space` / 클릭 | 화면 탭 |

- **공격**은 3연속 콤보. 마지막 타(피니셔)는 사거리·데미지·넉백이 강화됩니다.
- **회피 구르기**는 짧은 **무적 프레임**을 제공. 적 돌진·투사체 회피의 핵심.
- **필살기(커피 스매시)**는 적을 때려 모은 **SP**가 가득 찼을 때 발동하는 광역 충격파.

## 🕹️ 게임 개요 & 핵심 루프

탑다운 아레나에서 밀려오는 빌런 웨이브를 **실시간 액션**으로 전멸시키면 스테이지가 클리어됩니다.

```
이동 → 공격/콤보 → 회피(무적) → 필살기 → 웨이브 전멸 → 스토리 비트 → 레벨업/회복 → 다음 구역
```

- **성장**: 적 처치 시 EXP 획득 → 레벨업 시 최대 HP·ATK 상승 + 즉시 일부 회복.
- **드롭 아이템**: ☕ 커피(HP 회복), 🥤 에너지드링크(일정 시간 ATK 대폭 상승).
- **스테이지 클리어**시 HP 일부 회복 후 다음 구역으로.
- 사망 시 **이어하기(현재 구역부터)** / 처음부터 / 타이틀 선택. 컨티뉴 횟수는 최종 등급에 반영됩니다.

## 🗺️ 스테이지 구성 (판교역 → 위메이드플레이, 총 7구역)

| # | 구역 | 분위기 | 주요 빌런 |
|---|------|--------|-----------|
| 1 | 판교역 개찰구 | 튜토리얼, 파란 지하철 톤 | 잔액부족 게이트, 밀치는 인파 |
| 2 | 지하 환승 통로 | 어두운 미로 | 밀치는 인파, 길잃은 환승객, 역주행 에스컬레이터 |
| 3 | 지상 출구 계단 | 눈부신 지상 | 길잃은 환승객, 킥보드, 에스컬레이터 |
| 4 | 죽음의 횡단보도 | 신호 사거리 | 무단질주 킥보드, 변덕쟁이 신호등 |
| 5 | 카페거리 | 커피·알림 | 푸시 알림 떼, 커피 대기줄, 신호등 |
| 6 | 위메이드플레이 로비 | 회사 입구 | 출입증 인식오류(포탑), 커피 대기줄, 알림 |
| 7 | 최상층 · 보스전 | 어둠의 사무실 | **보스: 먼데이 모닝** |

스테이지가 진행될수록 적의 HP·공격력·이동속도 배율이 상승하여 **자연스러운 난이도 곡선**을 형성합니다.

## 👾 빌런(적) 디자인 — 각자 다른 AI 패턴

| 빌런 | 패턴 / 공략 |
|------|-------------|
| 💳 **잔액부족 게이트** | 느리게 추격. 기본 학습용. |
| 🚶 **밀치는 인파** | 빠르게 떼로 몰려옴. 넉백은 잘 되지만 수가 많음. |
| ❓ **길잃은 환승객** | 배회하다 **조준 후 돌진**. 돌진 예비동작을 보고 피하기. |
| 🔃 **역주행 에스컬레이터** | 수평 라인 **왕복**. **넉백 무효** — 밀 수 없으니 피해라. |
| 🛴 **무단질주 킥보드** | **예고 후 직선 대시**. 대시 직전 멈칫할 때가 기회. |
| 🚦 **변덕쟁이 신호등** | **빨간불=무적+투사체 발사**, **초록불=피격 가능**. 초록일 때만 딜. |
| 🔔 **푸시 알림 떼** | 맴돌며 투사체 사격. 약하지만 성가심. |
| ☕ **끝없는 커피 대기줄** | 느린 **탱커**, 높은 HP. |
| 🪪 **출입증 인식오류** | 고정 **포탑**, 사방으로 링 탄막. |
| 👿 **먼데이 모닝 (보스)** | 3페이즈 다단 인카운터 (아래). |

### 최종 보스 「먼데이 모닝」 — 3페이즈
1. **데드라인** — 추격 + 3방향 "마감" 사격, 간헐적 돌진.
2. **긴급회의** — 잡몹 **소환** + 링 폭발 탄막.
3. **알림 폭탄** — 회전 **나선 탄막** + 조준 산탄. 가장 빠르고 공격적.

각 페이즈 전환 시 잠깐 무적 + 화면 연출(플래시·흔들림)이 발생합니다.

## 🎇 연출 & 게임 필 ("손맛")

- **화면 흔들림**(피격/필살기/보스 연출), **화면 플래시**, **비네트**, **CRT 스캔라인**
- **데미지 숫자**(크리티컬 강조), **콤보 강화 타격**, **넉백**, **히트 스파크 파티클**
- **피격 무적 프레임 + 깜빡임**, 적 **히트 플래시**, 사망 파티클 폭발
- **WebAudio 8bit 사운드**: 효과음 전부 코드 신디사이즈 + 필드/보스 **루프 BGM** (🔊 버튼으로 음소거)
- **드래곤 퀘스트풍 UI**: 타자기 대사창, 클래식 프레임, 픽셀 커서, 레트로 HUD
- 인트로/스테이지 전환/보스 등장/엔딩까지 **하나의 완결된 여정**으로 연출

## ⚖️ 밸런싱 / 설계 의도

- **공정한 학습형 난이도**: 초반은 느린 추격형으로 조작을 익히고, 구역이 진행될수록 돌진·탄막·페이즈 기믹이 추가됨. 첫 시도 클리어는 어렵되 패턴을 학습하면 반드시 깰 수 있도록 설계.
- **회피 중심 액션**: 무적 프레임 회피가 핵심. 탱킹보다 **패턴 읽기 + 회피 + 반격**이 유리하도록 접촉/투사체 피해와 무적 시간을 튜닝.
- **성장 보상 루프**: 처치→EXP→레벨업(HP/ATK+회복)로 진행할수록 강해지는 손맛. 드롭 아이템으로 순간 파워스파이크 제공.
- **모든 튜닝 값은 [`game/constants.ts`](game/constants.ts) 한 곳에 집약** — 밸런싱 조율이 쉽도록 분리.

## 📁 프로젝트 구조

```
PanggyoQuest/
├─ app/
│  ├─ layout.tsx          # 루트 레이아웃 / 메타데이터
│  ├─ page.tsx            # <Game /> 마운트
│  └─ globals.css         # Tailwind + 레트로/CRT 스타일
├─ components/
│  ├─ Game.tsx            # 화면 상태머신 + 캔버스 + 오버레이 오케스트레이션
│  ├─ Hud.tsx             # 전투 HUD (HP/SP/EXP/스테이지/보스바)
│  ├─ MessageBox.tsx      # 드퀘풍 타자기 대사창
│  ├─ TouchControls.tsx   # 모바일 가상 조이스틱 + 액션 버튼
│  └─ screens.tsx         # 타이틀/일시정지/클리어/게임오버/승리 화면 + 등급 계산
├─ game/
│  ├─ engine.ts           # ★ 게임 엔진: 엔티티/전투/AI/렌더/이펙트/루프
│  ├─ constants.ts        # 밸런싱·튜닝 상수 (한 곳에서 조율)
│  ├─ types.ts            # 공용 타입
│  ├─ enemies.ts          # 빌런 정의(스탯/설명)
│  ├─ stages.ts           # 스테이지 구성·웨이브·스토리 대사
│  ├─ input.ts            # 키보드/터치 입력 관리
│  ├─ audio.ts            # WebAudio 레트로 사운드 신디사이저
│  └─ utils.ts            # 수학/유틸
├─ next.config.mjs        # output: 'export' (정적 배포)
├─ tailwind.config.ts
└─ tsconfig.json
```

**설계 원칙**: 전투 루프는 `GameEngine`(Canvas)가 단독 소유하고, React는 화면 전환·오버레이·HUD만 담당합니다. 둘 사이는 콜백(`EngineEvents`)과 `HudState`로 느슨하게 연결되어, 게임 로직과 UI가 명확히 분리되어 있습니다.

## 🏆 클리어 등급

플레이 타임과 컨티뉴 횟수로 **S / A / B / C** 등급이 결정됩니다.
- **S**: 무사고(컨티뉴 0) + 7분 이내 — 완벽한 정시 출근!
- **A**: 컨티뉴 ≤1 + 11분 이내
- **B**: 컨티뉴 ≤3
- **C**: 그 외 (그래도 출근은 출근)
